# 対戦テトリス 攻撃・防御システム 仕様書

## 概要

本ドキュメントは、対戦テトリスにおける**攻撃力の計算**・**お邪魔ブロックの保留と猶予**・**相殺処理**・**Firebaseを用いたデータ同期**の実装仕様を定義する。

---

## 1. 攻撃力の計算

### 1-1. 消去ライン数による攻撃力

| 消去ライン数 | 攻撃力（送るライン数） |
|:-----------:|:--------------------:|
| 1ライン      | 0                    |
| 2ライン      | 1                    |
| 3ライン      | 2                    |
| 4ライン（テトリス） | 4              |

### 1-2. T-Spinボーナス

T-Spinが成立した場合、通常の消去ライン数による攻撃力に加えてボーナスを加算する。

| T-Spinの種類     | ボーナス攻撃力 |
|:---------------:|:-----------:|
| T-Spin Single   | +2          |
| T-Spin Double   | +4          |
| T-Spin Triple   | +6          |

### 1-3. REN（連鎖）ボーナス

ラインを消し続けた連続回数（REN数）に応じて、攻撃力に加算する係数を設定する。

| REN数 | 加算ボーナス |
|:-----:|:-----------:|
| 1     | +0          |
| 2     | +1          |
| 3     | +1          |
| 4     | +2          |
| 5     | +2          |
| 6以上  | +3          |

> **ロジック補足：** ラインを1枚も消さずにミノを設置した場合、RENカウンターは0にリセットされる。

### 1-4. 最終攻撃力の計算式

```
最終攻撃力 = (消去ライン数による攻撃力) + (T-Spinボーナス) + (RENボーナス)
```

## 2. お邪魔ブロックの「保留」と「猶予」

攻撃を即座に盤面へ反映させると一瞬でゲームオーバーになるため、**攻撃キュー（待ち行列）**を介した保留処理を行う。

### 2-1. 攻撃キューの仕様

- 相手から攻撃が届いた際、即座に盤面へ反映せず、`pendingAttack`（整数値）に加算する。
- `pendingAttack` の値は常に画面端の**警告ゲージ**として可視化する。

### 2-2. 警告ゲージの表示

- 盤面の左端または右端に縦ゲージとして表示する。
- 溜まっているライン数に応じてゲージが伸びる。
- ゲージの色は攻撃量に応じて変化させることを推奨する（例：緑→黄→赤）。

### 2-3. 盤面への反映タイミング

お邪魔ブロックは**ミノを設置して、次のミノがスポーンする瞬間**に以下のフローで処理する。

```
ミノ設置
  └─→ 相殺処理を実行（後述のセクション3参照）
        └─→ 残った pendingAttack 分だけ盤面下部にお邪魔ブロックを挿入
              └─→ 次のミノをスポーン
```

> **猶予タイマー：** ラインを消去している間（消去アニメーション中）は、攻撃のせり上がりタイマーをリセットすることで、公式の挙動に近い猶予感を実現できる。

## 3. 相殺（打ち消し）の実装

### 3-1. 相殺ロジック

自分に攻撃が溜まっている状態でラインを消した場合、**相手に攻撃を送る前に、自分の `pendingAttack` を優先的に減算する**。

```
攻撃力 = 計算した最終攻撃力
pendingAttack（自分） > 0 の場合：
  相殺量 = min(攻撃力, pendingAttack)
  pendingAttack -= 相殺量
  攻撃力 -= 相殺量

攻撃力 > 0 であれば → 残った攻撃力を相手の pendingAttack に加算
```

### 3-2. 相殺の具体例

| 自分の pendingAttack | 自分の攻撃力 | 結果（自分側） | 結果（相手側） |
|:-------------------:|:----------:|:------------:|:------------:|
| 3                   | 2          | pendingAttack = 1 | 変化なし |
| 2                   | 2          | pendingAttack = 0 | 変化なし |
| 1                   | 4          | pendingAttack = 0 | +3        |
| 0                   | 4          | 変化なし          | +4        |

## 4. お邪魔ブロックの生成仕様

### 4-1. 穴の位置ルール

1回の攻撃で挿入される複数行のお邪魔ブロックは、**すべて同じ列に穴を配置する**。

```
例：3ライン攻撃の場合

 □□□□□□□□□□  ← 穴は列5
 □□□□□□□□□□  ← 穴は列5（同じ列）
 □□□□□□□□□□  ← 穴は列5（同じ列）
```

> **バランス上の理由：** 穴の位置を統一することで、受け手が穴を掘り続けることでカウンター（逆転）が可能になり、ゲームバランスが向上する。

### 4-2. 穴の列の決定

- 穴の列はランダムに決定する（毎回の攻撃ごとに再抽選）。
- ただし、前回の攻撃と同じ列にならないよう制御することを推奨する（連続して同じ列にならない仕様）。

## 5. Firebaseを用いたデータ同期

### 5-1. データ構造

```
/rooms/{roomId}/
  players/
    {playerId}/
      board: Array        // 盤面の状態（20×10の2次元配列）
      pendingAttack: Number  // 保留中の攻撃ライン数
      score: Number
      ren: Number         // 現在のRENカウンター
```

### 5-2. 攻撃を送る側の処理

1. ラインを消去した瞬間に最終攻撃力を計算する。
2. 自分の `pendingAttack` との相殺処理を行う。
3. 残った攻撃力が1以上であれば、**相手の `pendingAttack` に加算**する。

```javascript
// 相手の pendingAttack に攻撃力を加算
const opponentRef = ref(db, `/rooms/${roomId}/players/${opponentId}/pendingAttack`);
runTransaction(opponentRef, (current) => (current || 0) + attackPower);
```

### 5-3. 受ける側の処理

`onValue` で自分の `pendingAttack` の変化を監視し、警告ゲージをリアルタイム更新する。

```javascript
const myPendingRef = ref(db, `/rooms/${roomId}/players/${myId}/pendingAttack`);
onValue(myPendingRef, (snapshot) => {
  const pending = snapshot.val() || 0;
  updateWarningGauge(pending);  // ゲージUI更新
});
```

### 5-4. ミノ設置時の盤面反映フロー

```
1. ラインの消去判定・相殺処理を実行
2. pendingAttack の残量を取得
3. pendingAttack > 0 なら、盤面配列の下部に穴あきお邪魔ブロックを挿入
4. pendingAttack を 0 にリセット
5. 更新後の board と pendingAttack を Firebase に書き込む
6. 次のミノをスポーン
```

---

## 6. 処理フロー全体図

```
[ミノ設置]
    │
    ▼
[ライン消去判定]
    │
    ├─ 消去あり ─→ [攻撃力計算（ライン数 + T-Spin + REN）]
    │                   │
    │              [相殺処理]
    │              ├─ pendingAttack > 0 → pendingAttack を減算
    │              └─ 攻撃力が余った → 相手の pendingAttack に加算（Firebase書き込み）
    │
    ├─ 消去なし ─→ [RENカウンターをリセット]
    │
    ▼
[pendingAttack 残量を確認]
    │
    ├─ 残量 > 0 → [盤面下部にお邪魔ブロックを挿入]
    │              └─ pendingAttack = 0 にリセット
    │
    ▼
[盤面・状態を Firebase に書き込み]
    │
    ▼
[次のミノをスポーン]
```
